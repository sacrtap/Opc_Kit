# 变更记录条目模板

update 意图时，使用此模板生成变更记录条目。

## 条目格式

| 日期 | 版本 | 修改人 | 修改内容 |
|------|------|--------|---------|
| {YYYY-MM-DD} | {版本号} | {修改人} | {修改内容，动词开头，每点一句话} |

## 版本号自增规则

| 变更级别 | 规则 | 示例 |
|---------|------|------|
| 小改（文案修订、错别字、格式调整） | +0.0.1 | v1.5.0 → v1.5.1 |
| 中改（新增/修改功能描述、调整交互逻辑、修改验收标准） | +0.1.0 | v1.5.0 → v1.6.0 |
| 大改（增删 F-x.x 功能、表结构变更、核心流程重构） | +1.0.0 | v1.5.0 → v2.0.0 |

## 修改内容格式

动词开头，每点一句话，按影响范围排序：

```
新增 F-2.13 收藏夹分享功能，支持只读/编辑权限控制
更新 F-2.6 自动填充逻辑，增加竞态处理分支
修改成功指标：自动填充采纳率从 >80% 调整为 >85%
```

## 本次更新关键说明

在变更记录之后、第 1 章之前，添加一段自然语言说明：

```markdown
## 本次更新关键说明

本轮变更主要围绕 [变更主题]，涉及 [N] 个功能模块的调整。
核心设计意图：[为什么做这个变更，解决什么问题]。
影响范围：[影响哪些章节/功能/用户流程]。
与上版本的差异：[简要对比，帮助评审快速理解变更]。
```

update 意图时此段落必须存在，不可跳过。

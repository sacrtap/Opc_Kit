# Mermaid 流程图常用片段库

以下片段可直接复制修改，用于 PRD 第 5 章流程图。

## 片段 1：API 调用 + 判断分支

参考模式：F-2.1 GPS 逆地理编码

```mermaid
flowchart TD
    A[触发事件] --> B[调用外部 API<br/>服务名称]
    B --> C{API 结果}
    C -->|成功| D[获取数据字段]
    C -->|失败| E[标记失败状态]
    C -->|超时| E
    D --> F[写入数据库/返回前端]
    E --> F
```

**要点**：
- 判断节点必须有成功/失败/超时三分支
- 超时与失败合并到同一处理路径
- 超时时间应在交互说明中定义（如 >10s）

## 片段 2：并行处理 + 汇合

参考模式：F-2.1 GPS + F-2.2 LLM 并行解析

```mermaid
flowchart TD
    A[触发事件] --> B{并行调用}
    B --> C[服务 A<br/>API 名称]
    B --> D[服务 B<br/>API 名称]
    C --> E{A 结果}
    D --> F{B 结果}
    E -->|成功| G[获取 A 数据]
    E -->|失败/超时| H[A 数据为空]
    F -->|成功| I[获取 B 数据]
    F -->|失败/超时| J[B 数据为空]
    G --> K[汇合写入]
    H --> K
    I --> K
    J --> K
    K --> L[更新状态]
```

**要点**：
- 并行分支数量不超过 4 个，超出建议拆子流程
- 汇合节点明确说明如何处理部分失败

## 片段 3：状态机流转

参考模式：parse_status 状态流转

```mermaid
flowchart TD
    A[初始状态 pending] --> B{处理结果}
    B -->|A 成功 + B 成功| C[success]
    B -->|A 成功 + B 失败| D[partial]
    B -->|A 失败 + B 成功| D
    B -->|A 失败 + B 失败| E[failed]
    C --> F[前端行为：自动填充全部]
    D --> G[前端行为：填充可用部分]
    E --> H[前端行为：提示手动操作]
```

**要点**：
- 状态值必须与数据表 ENUM 定义一致
- 每个终态对应明确的前端/后端行为

## 片段 4：数据库读写 + 条件分支

参考模式：F-2.11 变更比对写入

```mermaid
flowchart TD
    A[触发事件] --> B[读取用户最终值]
    B --> C[读取系统原始值]
    C --> D{比对判断}
    D -->|完全一致| E[change_type = accepted]
    D -->|部分修改| F[change_type = modified]
    D -->|全部手动| G[change_type = manual]
    D -->|部分接受| H[change_type = accepted_partial]
    E --> I[写入日志表]
    F --> I
    G --> I
    H --> I
```

**要点**：
- 比对逻辑必须在交互说明中给出伪代码
- 写入操作标明表名和关键字段
